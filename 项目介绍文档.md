# XU-News-AI-RAG：个性化新闻智能知识库
## 项目介绍文档

---

## 1. 项目概述

### 1.1 项目名称
**XU-News-AI-RAG：个性化新闻智能知识库**

### 1.2 项目简介
XU-News-AI-RAG 是一个基于 RAG（检索增强生成）技术的个性化新闻智能知识库系统。系统通过自动化新闻采集、智能存储、语义检索和知识分析，为用户提供高效的新闻信息服务。

### 1.3 项目背景
在信息爆炸的时代，用户面临着海量新闻信息难以有效获取和管理的挑战。传统的新闻获取方式存在信息分散、检索效率低、缺乏个性化推荐等问题。本项目旨在通过AI技术构建一个智能新闻知识库系统，帮助用户更高效地获取、管理和检索新闻信息。

---

## 2. 核心功能

### 2.1 自动化新闻采集
- **定时任务**：每小时自动采集新闻
- **多源采集**：支持RSS订阅、网页抓取
- **规范遵守**：严格遵守robots.txt协议，控制访问频率
- **自动入库**：采集的新闻自动向量化并存储到知识库

### 2.2 智能知识库管理
- **文件上传**：支持多种格式（txt、xlsx、csv、md等）
- **知识库分类**：支持创建多个知识库，分类管理
- **元数据管理**：支持编辑标签、来源等元数据
- **文件操作**：支持查看、编辑、删除、移动文件

### 2.3 语义检索
- **自然语言查询**：支持自然语言提问
- **向量检索**：基于FAISS向量数据库的快速检索
- **结果重排**：使用CrossEncoder对结果进行相关性重排
- **智能回复**：使用大语言模型生成智能回复
- **联网查询**：知识库未匹配时自动触发联网搜索

### 2.4 数据分析
- **关键词提取**：自动提取文档关键词（Top10）
- **聚类分析**：基于关键词相似度的快速聚类
- **可视化展示**：图表展示关键词分布和聚类结果
- **分析报告**：生成分析结论和洞察

### 2.5 用户认证
- **用户注册**：邮箱验证码注册
- **用户登录**：JWT Token认证
- **权限管理**：基于Token的访问控制

## 3. 技术亮点

### 3.1 AI技术栈
- **嵌入模型**：all-MiniLM-L6-v2（768维向量）
- **重排模型**：ms-marco-MiniLM-L-6-v2（相关性排序）
- **大语言模型**：qwen2.5:4b（通过Ollama本地部署）
- **向量数据库**：FAISS（高性能向量检索）

### 3.2 架构设计
- **前后端分离**：React + Flask
- **RESTful API**：标准化API接口
- **ORM框架**：SQLAlchemy，支持SQLite和MySQL
- **模块化设计**：代码结构清晰，易于维护

### 3.3 性能优化
- **向量检索**：FAISS索引，检索速度 < 100ms（10万条数据）
- **异步任务**：分析任务后台执行，不阻塞用户操作
- **结果缓存**：常用查询结果缓存（可选）

### 3.4 技术选型说明
- **未使用LangChain**：采用sentence-transformers直接实现，性能更优，代码更直观
- **数据库灵活**：支持SQLite（开发）和MySQL（生产），通过环境变量切换

## 4. 系统架构

### 4.1 整体架构
```
前端 (React) → API (Flask) → 业务逻辑层 → 数据存储层
                                    ├── SQLite/MySQL
                                    ├── FAISS
                                    └── Ollama
```

### 4.2 数据流
1. **新闻采集**：RSS/网页 → 爬虫 → 数据清洗 → 向量化 → FAISS存储
2. **语义检索**：用户提问 → 向量化 → FAISS检索 → 重排序 → 大模型生成回复
3. **数据分析**：文档集合 → 关键词提取 → 聚类分析 → 可视化展示
